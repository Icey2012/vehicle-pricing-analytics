# Project Proposal: Vehicle Competitive Intelligence & Pricing Analytics

**Version**: 1.0  
**Date**: April 13, 2026  
**Status**: Ready for execution

---

## 1. Executive Summary

This project builds a data-driven pricing analytics framework for the automotive industry. It enables automatic competitor identification and quantifies how vehicle configurations translate into market pricing. The final output is an interactive dashboard suitable for portfolio presentation in job interviews.

---

## 2. Business Value

| Stakeholder | Value Proposition |
|-------------|-------------------|
| OEMs / Product Managers | New vehicle pricing, feature strategy, competitive response |
| Consumers | Purchase decision support, feature value assessment |
| Job Interview (Portfolio) | End-to-end data capability + industry domain knowledge |

---

## 3. Project Scope

### 3.1 Vehicle Coverage
- **Included**: ICE (internal combustion engine) + NEV (BEV / PHEV / EREV)
- **Excluded**: Commercial vehicles, motorcycles, models discontinued >5 years

### 3.2 Data Sources

| Layer | Source | Notes |
|-------|--------|-------|
| Core specs | Autohome + Dongchedi | Cross-validated for accuracy |
| Sales volume | Dongchedi monthly / CPCA | Insight module only, not in pricing model |
| Brand tier | Manual labeling | Premium / Mass / Economy |
| Marketing cases | Public sources | Qualitative analysis only |

---

## 4. Feature Definition

### 4.1 Target Variable (Y)

| Field | Description |
|-------|-------------|
| MSRP (10k RMB) | Manufacturer's suggested retail price |

### 4.2 Core Features (X — mandatory)

| Field | Type | Description |
|-------|------|-------------|
| Powertrain | Categorical | BEV / PHEV / EREV / ICE |
| Range (km) | Numeric | CLTC/WLTC for NEV; fuel tank + consumption for ICE (optional) |
| Battery capacity (kWh) | Numeric | NEV only; ICE = null |
| Battery chemistry | Categorical | NCM / LFP / N/A |
| Displacement (L) | Numeric | ICE core; NEV = null |
| Engine power (kW) | Numeric | ICE |
| Motor power (kW) | Numeric | NEV |
| 0-100 km/h (sec) | Numeric | Universal |
| Length (mm) | Numeric | Universal |
| Wheelbase (mm) | Numeric | Universal |
| Brand (raw) | Categorical | Original brand name |
| Brand tier | Categorical | Premium / Mass / Economy (manually labeled) |
| Vehicle segment | Categorical | A / B / C / D / SUV / MPV |

### 4.3 Smart Hardware Features (recommended)

| Field | Notes |
|-------|-------|
| LiDAR | Binary (yes/no) |
| ADAS level | L1 / L2 / L2+ (data quality varies) |

### 4.4 Non-Modeling Module (standalone insights)

| Field | Purpose |
|-------|---------|
| Monthly sales volume | Price-sales insight; not in regression |
| Marketing case studies | 2-3 vehicles, qualitative only |

## 5. Technical Architecture

### 5.1 Pipeline Overview

```
Scraping → Cleaning (pandas) → Modeling (sklearn) → Dashboard (Streamlit)
```

### 5.2 Tech Stack

| Component | Tools / Libraries |
|-----------|-------------------|
| Scraping | requests, BeautifulSoup, selenium (standby) |
| Data processing | pandas, numpy, re |
| Modeling | scikit-learn (clustering + regression), XGBoost |
| Visualization | matplotlib, seaborn, plotly |
| Interactive dashboard | Streamlit |
| Environment | Python 3.9+, Jupyter, VS Code |
| Version control | Git + GitHub |

## 8. Risks & Mitigation

| Risk | Probability | Mitigation |
|------|-------------|------------|
| Anti-scraping blocking | Medium | Rate limiting, rotating User-Agent, fallback to selenium |
| High missing rate in fields | Medium | Imputation / drop threshold (>30% missing = drop) |
| ICE vs NEV feature mismatch | Medium | Separate models or unified encoding (null → -1 / "N/A") |
| Timeline overrun | Low | Week 3 as buffer; can drop non-essential features |
